# Hybrid FAQ Retrieval Benchmark

Воспроизводимый стенд для сравнения вариантов поиска по FAQ-корпусу: поиск BM25, векторный поиск, объединение сигналов и переранжирование.

## Назначение

По пользовательскому вопросу находить наиболее релевантный ответ в корпусе FAQ-документов и сравнивать качество разных стратегий поиска в единых условиях.

## Архитектура

```text
FAQ-корпус
    │
    ▼
Подготовка документов и запросов
    │
    ├─ поиск BM25
    ├─ векторный поиск
    └─ объединение сигналов
            │
            ▼
      Переранжирование
            │
            ▼
      Расчёт метрик качества
            │
            └─ API поиска
```

## Что реализовано

- локальный корпус FAQ и набор запросов;
- несколько режимов поиска в одном стенде;
- единый сценарий оценки;
- сохранение сводных результатов запуска;
- API для локальной проверки качества поиска.

## Структура репозитория

- `configs/` — конфигурации экспериментов;
- `data/demo_faq/` — демонстрационный корпус и эталонные соответствия;
- `examples/` — примеры запросов;
- `reports/` — отчёты запусков;
- `scripts/` — сценарии запуска;
- `src/faq_bench/` — основной код;
- `tests/` — тесты;
- `Dockerfile`, `Makefile` — воспроизводимый запуск.

## Оценка качества

Основные метрики:

- `Recall@k`
- `MRR@k`
- `nDCG@k`
- `HitRate@k`
- задержка ответа: `latency_ms_p50`, `latency_ms_p95`

Сравнение режимов:

- поиск BM25;
- векторный поиск;
- объединение сигналов;
- объединение сигналов с переранжированием.

## Результаты

Демонстрационный корпус: 6 запросов, 8 документов. Метрики при `k = 5` после локального прогона `python scripts/run_benchmark.py` с конфигурациями из репозитория.

| Режим (config) | Recall@5 | MRR@5 | nDCG@5 | HitRate@5 | latency_ms_p50 | latency_ms_p95 |
|----------------|----------|-------|--------|-----------|----------------|----------------|
| BM25 (`configs/bm25.yaml`) | 1.0 | 1.0 | 1.0 | 1.0 | 0.029 | 0.053 |
| Векторный поиск (`configs/dense.yaml`) | 1.0 | 1.0 | 1.0 | 1.0 | 6.19 | 7.75 |
| Объединение + переранжирование (`configs/hybrid.yaml`) | 1.0 | 1.0 | 1.0 | 1.0 | 29.17 | 60.53 |

На малом корпусе метрики часто достигают 1.0; для выводов о качестве на реальных данных требуется свой корпус и отдельный прогон.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python scripts/run_benchmark.py --config configs/bm25.yaml
python scripts/run_benchmark.py --config configs/dense.yaml
python scripts/run_benchmark.py --config configs/hybrid.yaml
```

## Ограничения

- эффект векторного поиска зависит от выбранной модели представления текста;
- эффект переранжирования зависит от качества исходного набора кандидатов;
- для переноса в промышленный контур нужен корпус большего объёма и журналирование качества на реальных запросах.
