# Hybrid FAQ Retrieval Benchmark

## Кратко
Стенд сравнения подходов к FAQ-поиску в одинаковых условиях: BM25, векторный поиск, объединение результатов и дополнительное переранжирование. Репозиторий помогает понять, где выигрывает полнота поиска, а где растёт качество верхних позиций выдачи.

## Задача
FAQ-корпус содержит короткие вопросы и ответы, поэтому ни один режим поиска не является универсальным. Лексический поиск хорошо работает на точных формулировках, а векторный — на перефразированных запросах. В проекте сравниваются оба режима и их объединение по единому протоколу оценки.

## Что улучшено
- объединение BM25 и векторного поиска повышает полноту выдачи;
- дополнительное переранжирование улучшает качество верхних позиций;
- единый стенд делает сравнение режимов воспроизводимым и сопоставимым.

## Архитектура
```text
[FAQ-корпус]
      |
      v
[подготовка индексов]
      |
      +----------------------+----------------------+
      |                                             |
      v                                             v
[BM25]                                     [векторный поиск]
      |                                             |
      +----------------------+----------------------+
                             |
                             v
               [объединение результатов (RRF)]
                             |
                             v
                  [дополнительное переранжирование]
                             |
                             v
               [оценка качества на отложенной выборке]
```

## Структура репозитория
```text
src/       основная логика поиска и оценки
scripts/   сценарии запуска
reports/   результаты и отчёты, если сохранены
data/      данные для сравнения
```

## Критерии качества
- Recall@K
- MRR@K
- nDCG@K
- HitRate@K
- задержка ответа

## Ключевой вывод
Лексический поиск и векторный поиск решают разные подзадачи. Их объединение обычно повышает полноту выдачи, а дополнительное переранжирование улучшает качество верхних позиций списка. Итоговый режим выбирается по балансу качества и задержки.

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m src
```

## Ограничения
- качество векторного поиска зависит от выбранной модели представлений;
- выигрыш от объединения режимов зависит от разнообразия FAQ-корпуса;
- дополнительное переранжирование увеличивает вычислительную стоимость запроса.
