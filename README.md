# Hybrid FAQ Retrieval Benchmark

Сравнение подходов к поиску ответов в FAQ-корпусе: `BM25`, dense retrieval, их объединение и дополнительное переранжирование в едином контуре оценки.

## Задача
Нужно сравнить несколько подходов к поиску ответов в FAQ и понять, какой из них лучше удерживает релевантные ответы в верхней части выдачи.

## Архитектура решения
```text
FAQ-корпус
   │
   ▼
Подготовка индексов
   │
   ├──► BM25
   │
   ├──► Dense retrieval
   │
   ▼
Объединение результатов
   │
   ▼
Переранжирование
   │
   ▼
Оценка качества
```

## Структура репозитория
- `configs/` — параметры запуска;
- `data/demo_faq/` — демонстрационный корпус;
- `src/faq_bench/` — основной код;
- `scripts/` — запуск сравнения;
- `reports/` — итоговые таблицы и графики;
- `PROJECT_SPEC.md` — постановка задачи;
- `tests/` — проверка компонентов.

## Метрики
- `Recall@K`
- `MRR@K`
- `nDCG@K`
- `HitRate@K`
- задержка ответа (`p50`, `p95`)

## Результаты
Папка `reports/` на публичной странице пока пуста, поэтому итоговые значения нужно получить запуском сценариев оценки и записать в `reports/metrics_summary.md` и `reports/metrics_summary.json`.

В README должны появиться:
- сравнение `BM25`, dense retrieval, объединённого варианта и варианта с переранжированием;
- изменение `Recall@K`, `MRR@K`, `nDCG@K`;
- задержка каждого контура.

Формат записи:
- `Объединение BM25 и dense retrieval повысило Recall@10 с ... до ...`
- `Переранжирование повысило MRR@10 с ... до ...`
- `Задержка полного контура составила ... мс`

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/run_benchmark.py
```

## Ограничения и развитие
- Качество dense retrieval зависит от выбранной модели векторизации;
- выигрыш от переранжирования зависит от качества кандидатов;
- следующий шаг — автоматизировать сохранение результатов в `reports/`.
